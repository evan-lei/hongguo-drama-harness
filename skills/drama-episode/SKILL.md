# 分集剧本生成

## 何时使用

输入为已确认的**分集大纲**（`outline.md`），产出符合红果格式的**分集剧本**。

**不做什么**：不修改大纲情节点；不一次生成全剧（除非用户明确要求）——默认逐集或按批次（如 1–3 集）生成。

## 输入

- `projects/<剧名>/outline.md`（推荐）
- 指定集数范围（如「第 1–3 集」）；未指定时默认第 1–3 集

## 输出

每集保存为 `projects/<剧名>/episodes/ep<N>.md`，格式如下：

```markdown
# 第N集 · <集标题（可选）>

> 状态：剧本 | 日期：YYYY-MM-DD

第N集

N-1 日/夜 内/外 <地点>
人物：<角色列表>

【开场钩子】
△ <动作/环境描写>

<角色>（<语气>）：<台词>

N-2 ...

【结尾卡点】
△ <卡点画面>
```

## 格式规范（遵守 hongguo-platform）

| 要素 | 要求 |
|------|------|
| 场次头 | `N-M 日 内 地点` + `人物：` |
| 动作 | `△` 开头，具象、可拍摄 |
| 语气 | `()` 标注对白时的神态/动作 |
| 结构标注 | `【开场钩子】` `【转折】` `【结尾卡点】` |
| **正文字数** | **300–2500 汉字**（创服上传硬性规则）；Harness 生成目标 **600–900 汉字/集** |
| 台词 | 短句、口语化、有动机 |

### 正文字数定义

| 计入 | 不计入 |
|------|--------|
| 场次头、人物行 | `# 集标题` |
| `△` 动作描写 | `> 状态` 元数据 |
| 对白及 `()` 语气标注 | 文件顶部 Markdown |
| `【】` 结构标注 | |

生成后**必须**统计汉字数；< 300 自动扩写，> 2500 自动删减，无需用户确认。

### 红线

- ❌ 大段心理描写
- ❌ 文学化叙述、不可拍摄的抽象描写
- ❌ 人物无故消失或场景跳跃
- ✅ 每场戏有冲突或信息推进

## 生成规则

### 1. 忠于大纲

- 本集开场钩子、核心事件、结尾卡点与 outline 一致
- 可补充过渡场次，不得改变卡点含义

### 2. 单场戏结构

- 每场 1 个核心冲突
- 对白推动剧情，不写废话寒暄
- 开场 3–5 秒（约 2–4 行）必须进冲突

### 3. 批次策略

| 用户请求 | 做法 |
|----------|------|
| 未指定 | 生成第 1–3 集，供打磨前 3 集 |
| 指定 `第 N 集` | 只生成该集 |
| 指定 `第 A–B 集` | 按范围生成，每集单独文件 |

### 4. 写后自检

- [ ] **正文字数 300–2500 汉字？**（目标 600–900）
- [ ] 开场有钩子、结尾有卡点？
- [ ] 场次编号连续？
- [ ] 人物与大纲一致？

### 5. 字数不足时的扩写策略

低于 300 字或低于 Harness 目标 600 字时，按序扩写（不改情节点）：

1. 每场增加 1–2 句 △ 可拍摄动作
2. 冲突双方各加 1–2 轮短台词交锋
3. 可增 1 场过渡戏（同集内，不跳集）
4. 禁止用心理独白凑字数

## 工作流

```
1. 读取 outline.md + 确认集数范围
2. 读取 hongguo-platform 格式规范
3. 逐集生成，每集单独文件
4. 每集完成后做篇幅与卡点自检
5. 呈现给用户，建议跑 drama-qc
```
