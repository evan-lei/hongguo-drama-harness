# 短剧故事初稿 Prompt 丰富

## 何时使用

用户给出碎片输入（几个关键词、一句脑洞、模糊人设），需要产出**结构化故事初稿**，作为后续大纲 / 分集剧本的输入。

**不做什么**：不直接写完整分集剧本；不跳过故事核确认。

## 输入

用户可能提供：

- 题材关键词（如「重生复仇」「隐藏富豪」）
- 一句话脑洞
- 偏好情绪（爽 / 虐 / 甜）
- 目标受众
- 参考剧名（仅供结构借鉴，禁止洗稿）

若信息不足，**最多追问 3 个关键问题**（题材、核心反差、情绪类型），然后基于合理假设继续，并在输出中标注「待确认项」。

## 丰富规则

### 1. 短剧核心公式（必须体现）

- **人设极端化** — 身份反差拉满
- **冲突前置** — 第 1 集开局就要有事
- **情绪价值明确** — 观众追的是哪种爽/虐/甜
- **集集有钩子** — 前 3 集卡点必须具体
- **可拍摄** — 事件链用动作和冲突描述，不用心理独白

### 2. 人物设计

- 主角：欲望 + 障碍 + 反差，三者缺一不可
- 反派：动机合理，最好「不自知是反派」或「有苦衷」
- 配角：每个出场人物有功能，不写工具人

### 3. 故事线

- 主线只能有一条，B 故事服务于 A 故事
- 用「因此/但是」链验证因果：事件 1 **但是** 事件 2 **因此** 事件 3

### 4. 差异化

- 同题材要有**一个**清晰差异点（人设 / 时代 / 切入点 / 反转方向）
- 不照搬已有爆款桥段

### 5. 与平台规范对齐

产出后对照 `hongguo-platform` 质检速查；有冲突时以平台规范为准。

## 工作流

```
1. 读取用户碎片输入
2. 信息不足 → 追问 ≤3 个问题（或标注待确认项）
3. 按模板生成故事初稿
4. 自检：一句话梗概、前3集钩子、人物反差、主线清晰度
5. 呈现给用户确认，不自动进入大纲阶段
6. 用户确认后 → 建议保存到 projects/<剧名>/brief.md
```

## 示例

**用户输入**：「我想写一个女保镖隐藏身份给霸总当保姆的故事，要爽文」

**输出要点**（节选）：

- 一句话：王牌保镖伪装保姆潜入豪门，边护雇主边查清灭门真相
- 反差：表面唯唯诺诺的小保姆 / 实则一人能打十个的前特工
- 第 1 集钩子：入职第一天就撞见雇主被刺杀，她本能出手又强行装怂
- 第 1 集卡点：霸总捡起她「失手」掉出的弹壳，眼神变了
