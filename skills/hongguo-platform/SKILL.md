# 红果短剧平台规范与避坑

## 平台定位

红果短剧是字节跳动旗下短剧平台，创作入口为**红果短剧创作服务平台**。个人编剧可实名入驻，直接投稿故事大纲 / 分集大纲 / 完整剧本。

核心认知（来自官方「短剧编剧第一课」）：
- 短剧是**成年人的童话**，提供强烈情绪点和代入感
- 编剧要从「作家」转为**洞察用户需求的「产品经理」**
- 剧本不是文学作品，是**拍摄指南**

## 入驻与投稿

### 入驻条件

- 年满 18 周岁
- 本人身份证实名认证
- 本人实名手机号
- 本人名下银行卡（稿费结算）
- 一账号一身份（编剧 / 制作方二选一），实名绑定，禁止代投

### 投稿类型（由轻到重）

| 类型 | 适用场景 |
|------|----------|
| 故事大纲 | 新手试水、验证创意，反馈快 |
| 分集大纲 | 主线确认后，规划节奏与卡点 |
| 完整剧本 | 大纲过审后深化，进入制作流程 |

**建议路径**：先投故事大纲 → 拿编辑反馈 → 再写分集大纲 / 完整剧本。

### 平台评估维度

- 剧本质量
- 题材匹配度
- 市场潜力

## 剧本格式规范

### 标准场次头

```
第x集

1-1 日 内 林府大厅
人物：林雨欣、林母、养妹
```

| 要素 | 说明 |
|------|------|
| 集数 | `第x集` |
| 场号 | `场1-1` = 第1集第1场；同集多场递增 `1-2`、`1-3` |
| 时间 | `日` / `夜` |
| 空间 | `内` / `外` |
| 地点 | 具体场景名 |
| 人物 | 本场出场角色列表 |

### 正文标记

| 标记 | 用途 |
|------|------|
| `△` | 动作、环境、非对白叙述 |
| `()` | 对白时的语气、神态、小动作 |
| `【】` | 集内结构标注：开场钩子 / 转折 / 卡点（大纲阶段常用） |
| `【闪回】`…`【闪出】` | 回忆段落 |

### 篇幅要求（红果创服上传硬性规则）

> **投稿前以创服后台实时提示为准。** 以下为当前上传实测要求。

| 项目 | 要求 |
|------|------|
| **正文字数** | 每集 **300–2500 字**（仅计剧本正文，不含剧名/备注） |
| **正文范围** | 从 `第x集` 起，至本集最后一场戏止；含场次头、△ 动作、对白、结构标注 |
| **不计入** | 文件顶部 `# 标题`、`> 状态` 等 Markdown 元数据 |
| **Harness 目标** | 生成时瞄准 **600–900 字/集**（高于平台上限中位，留删减空间） |
| **红线** | ❌ 低于 300 字无法上传；❌ 超过 2500 字需删减 |

### 正文字数自检（生成后必跑）

```bash
python3 scripts/count-episode-chars.py projects/<剧名>/episodes
```

- 汉字数 < 300 → **必须扩写**后再上传
- 汉字数 > 2500 → **必须删减**
- 300–2500 且建议 < 900 → ✅ 可上传

### 篇幅参考（旧，已被上表取代）

- ~~每集约 600–1000 字~~ → 以创服 **300–2500 正文字数** 为准

### 格式红线

- ❌ 大段心理描写、文学化叙述
- ❌ 场景和人物前后不一致（上集站 A 处，下集无故消失）
- ❌ 缺少可拍摄的具象动作（「在房里发疯」→ 需写清动线）
- ✅ 每句话都能被导演、演员理解并执行

## 结构与节奏

### 单集框架

1. **开场 3–5 秒设钩** — 强冲突 / 强悬念 / 强反差
2. **中段推进** — 冲突升级、信息揭露
3. **结尾卡点** — 悬念留钩，驱动下一集

### 全剧四阶段（拉片三层次法第二层）

| 阶段 | 核心任务 |
|------|----------|
| 前 10 集 | 立人设、建冲突、抓观众 |
| 10–30 集 | 矛盾升级、情绪递进 |
| 30–60 集 | 反转、高潮铺垫 |
| 60 集至结局 | 收束、情绪爆点、余韵 |

### 短剧核心公式

- **人设极端化** — 身份反差要大
- **冲突前置** — 第 1 分钟有矛盾
- **情绪价值明确** — 爽 / 虐 / 甜 / 燃，观众知道在追什么
- **集集有钩子** — 每集结尾留悬念
- **台词可演** — 口语化、短句、竖屏快剪友好

## 平台政策（需持续关注）

> 政策会变，投稿前务必查看创服后台最新公告。

| 政策 | 要点 |
|------|------|
| 限投 | 单账号 30 天内签约剧本有上限（约 10 本），打击批量产能 |
| 查重 | AI 原创性验证，抄袭可封号 |
| AI 仿真人剧 | 部分剧本保底收紧，更看数据和精品化 |
| 预算方向 | 真人精品剧仍是重点，总预算仍在增长 |

AI 使用边界：**可辅助灵感、扩写、润色**；故事核、人设、关键转折须创作者把控。

## 避坑清单 ⚠️

### 创作层

| 坑 | 正确做法 |
|----|----------|
| 纯 AI 一键出稿就投 | 分层生成 + 人工精修故事核 |
| 支线太多、主线不清 | 一句话概括全剧核心，做减法 |
| 开场慢热、长铺垫 | 第 1 集前 30 秒抛核心设定或冲突 |
| 台词书面化、过长 | 短句、口语、推动剧情 |
| 角色动机不明 | 每句对白背后有动机 |
| 同质化桥段 | 在人设或切入点上做微创新 |

### 投稿层

| 坑 | 正确做法 |
|----|----------|
| 新手直接投 30 集全稿 | 先投故事大纲试水 |
| 格式混乱 | 先「写得清楚」再「写得惊艳」 |
| 一稿多投 / 代投 | 实名绑定，遵守限投 |
| 抄袭或洗稿热梗 | 自查原创性，差异化人设 |
| 忽视后台格式要求 | 投稿前对照后台最新模板 |

### 心态层

| 坑 | 正确做法 |
|----|----------|
| 「门槛低来钱快」 | 认知深度（市场敏锐度、逻辑力、抗压性）才是壁垒 |
| 追求产量不追求质量 | 10 本里 1–2 本精品 > 10 本平庸 |
| 不接编辑反馈 | 把改稿当常态，前 3 集反复打磨 |

## 质检速查（投稿前 30 秒）

```
- [ ] 一句话能说清全剧核心？
- [ ] 第 1 集开场有强钩子？
- [ ] 每集结尾有卡点？
- [ ] 人设反差足够极端？
- [ ] 主线清晰、支线已删减？
- [ ] 格式符合场次规范？
- [ ] 每集正文字数 300–2500 汉字？（`count-episode-chars.py`）
- [ ] 批量上传已导出且集标题为「第一集」中文数字？（`export-batch-upload.py`）
- [ ] 动作可拍摄、人物不丢？
- [ ] 台词短、可演、有动机？
```

## 剧本批量上传格式

> **适用范围**：创服后台「剧本批量上传」功能。  
> **不适用**：单集逐条上传（单集上传格式见上文场次规范）。

### 格式规则

| 规则 | 说明 |
|------|------|
| 章节标题 | 必须用中文数字：**第一集**、**第二集**……**第三十集** |
| 章节分隔 | 每集之间**空一行**（换行分隔） |
| 第零集 | 可选；用于承载**人物小传** + **剧本大纲**，标题写 **第零集** |
| 无元数据 | 不上传 `# 标题`、`> 状态` 等 Markdown 备注 |
| 场次编号 | 批量文件内仍用 `1-1` `1-2` 数字场号（与单集剧本一致） |

### 文件结构示例

```
第零集

人物小传：陈野
身份：汽修厂学徒 / 天才工程师
……

人物小传：苏棠
……

剧本大纲
起：……
承：……
转：……
合：……

第一集

1-1 日 内 振华汽修厂车间
人物：陈野、赵坤
……

第二集

2-1 夜 外 ……
……
```

### 第零集说明

- **可投可不投**：不需要人物小传/大纲时，可跳过第零集，不影响批量上传
- **Harness 默认**：导出时**包含**第零集（从 `brief.md` + `outline.md` 自动拼装）

### 导出命令

```bash
python3 scripts/export-batch-upload.py projects/<剧名>
# 输出：projects/<剧名>/upload-batch.txt

# 不含第零集：
python3 scripts/export-batch-upload.py projects/<剧名> --no-episode-zero
```

### 批量上传避坑

| 坑 | 正确做法 |
|----|----------|
| 用「第1集」阿拉伯数字 | 批量上传必须用 **第一集** 中文数字 |
| 多集粘在一起无空行 | 集与集之间保留空行 |
| 把 Markdown 元数据一并上传 | 只上传正文，去掉 `#` `>` 行 |
| 单集字数不足 300 | 导出前跑 `count-episode-chars.py` |
| 单集与批量格式混用 | 确认后台入口是「批量上传」再用本格式 |
