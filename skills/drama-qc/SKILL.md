# 短剧剧本质检

## 何时使用

对任意阶段稿件做**结构化质检**，产出可操作的修改清单。

支持：`brief.md` / `outline.md` / `episodes/ep*.md` / 用户粘贴的片段

## 输出

保存到 `projects/<剧名>/qc-<阶段>-<日期>.md` 或直接呈现：

```markdown
# 质检报告 · <剧名> · <阶段>

> 日期：YYYY-MM-DD
> 稿件：<文件路径>
> 结论：✅ 可投稿 / ⚠️ 需修改 / ❌ 不建议投稿

## 总评（3 句话以内）

## 评分明细

| 维度 | 得分 | 说明 |
|------|------|------|
| 故事核清晰度 | /10 | |
| 人设反差 | /10 | |
```

## 质检维度

### 1. 故事核（brief / 全阶段）

- 一句话能否说清全剧？
- 主线是否唯一、清晰？
- 差异化是否具体（不是「逆袭」这种空话）？

### 2. 人设反差

- 主角表/里身份反差是否足够极端？
- 反派动机是否合理？

### 3. 钩子与卡点

- 第 1 集开场是否在「前 30 秒」有强冲突？
- 每集结尾是否有悬念驱动下一集？
- 卡点是否可拍摄（具体画面，非抽象感受）？

### 4. 逻辑

- 事件链能否用「因此/但是」串通？
- 角色行为是否有动机？
- 有无 Deus ex machina（机械降神）？

### 5. 可拍摄性（episode 重点）

- 动作用 `△` 能写清吗？
- 人物场次是否一致？
- 有无大段不可拍的心理描写？

### 6. 台词

- 是否口语化、短句？
- 每句台词是否推动剧情或揭示性格？

### 7. 格式（outline / episode）

- 场次头、场号、人物列表是否齐全？
- **episode 正文字数是否在 300–2500 汉字？**（低于 300 或高于 2500 为 🔴 必改，自动扩写/删减）

### 8. 平台合规

对照 `hongguo-platform` 避坑清单与质检速查 8 项。

## 结论标准

| 结论 | 条件 |
|------|------|
| ✅ 可投稿 | 无必改项，均分 ≥ 7 |
| ⚠️ 需修改 | 有必改项，或均分 5–7 |
| ❌ 不建议投稿 | 故事核不清、严重逻辑漏洞、或同质化极高 |

## 工作流

```
1. 识别稿件阶段（brief / outline / episode）
2. 读取 hongguo-platform 避坑与速查清单
3. 按阶段适用维度逐项检查
4. 输出报告，必改项须具体可执行（不说「加强冲突」，要说「第 2 集结尾改为 X」）
5. 【自动修复】发现必改项 → 直接修订源稿件 → 复检 → 无需用户确认
```

## 自动修复（默认开启，无需用户确认）

质检发现 **🔴 必改项** 时，**不要停下来等用户确认**，按以下流程自动处理：

```
质检 → 列出必改项 → 立即修订源文件 → 写入 qc 报告（含修订记录）→ 复检 → 汇报最终结论
```

### 修订原则

| 原则 | 说明 |
|------|------|
| 只改必改项 | 🟡 建议改项不自动改，除非能零副作用合并 |
| 不改故事核 | 不修改用户已确认的人设、结局方向、核心反差 |
| 最小改动 | 能改一句不改一段，能改一集不改全剧 |
| 可执行 | 每条必改项必须对应到具体文件、具体集数/段落 |
| 复检闭环 | 修订后重新跑一遍质检；仍有必改项则继续修，直到 ✅ 或无法自动修（见下） |

### 源文件映射

| 阶段 | 修订目标 |
|------|----------|
| brief | `projects/<剧名>/brief.md` |
| outline | `projects/<剧名>/outline.md` |
| episode | `projects/<剧名>/episodes/ep<N>.md` |

### 常见必改项 → 自动修复规则

| 问题类型 | 自动修复动作 |
|----------|--------------|
| 梗概与分集矛盾 | 以分集为准统一梗概，或补充「赛场时间线」表消歧 |
| 重复情节（如泄密两次相同） | 拆为两阶段：早期草图泄密 → 后期核心数据泄密 |
| 伏笔无铺垫（如众筹零件突然出现） | 在前 3–5 集插入一句可拍摄铺垫 |
| 正文字数不足 300 | 按 `drama-episode` 扩写策略增补场次/台词/动作，不改情节点 |
| 正文字数超过 2500 | 删减重复台词与过渡场，保留钩子/卡点/核心冲突 |
| 合规缺口（砸车间无报警） | 在同集补充「报警 → 警方/行业封杀」，删除私刑线 |
| 赛场时间线不清 | 在 outline 增加「赛场时间线」表，标注每场的性质与结果 |
| 格式红线（episode） | 补场次头、△ 动作、卡点标注 |

### 无法自动修的情况（才询问用户）

- 故事核本身需改方向（结局、主角人设推翻）
- 两条必改项互相冲突，无法同时满足
- 用户稿件 `brief.md` 与 `outline.md` 核心设定根本矛盾

此时在 qc 报告中标注「需人工决策」，**仅此时**暂停并询问。

### qc 报告补充字段

自动修复后，报告须增加：

```markdown
## 自动修订记录
- [x] 必改项1：… → 已修订 outline.md 第N集 / 故事梗概
- [x] 必改项2：…

## 复检结论
> 修订后结论：✅ 可投稿 / ⚠️ 仍有问题
```
