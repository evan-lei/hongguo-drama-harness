# 短剧大纲生成

## 何时使用

输入为已确认的**故事初稿**（`brief.md`），产出红果投稿用的**故事大纲 + 分集大纲**。

**不做什么**：不写完整台词剧本；不修改已确认的故事核（除非标注逻辑风险并征得用户同意）。

## 输入

- `projects/<剧名>/brief.md`（推荐）
- 或用户粘贴的完整故事初稿

## 输出

保存到 `projects/<剧名>/outline.md`，结构如下：

```markdown
# <剧名> · 大纲

> 状态：大纲 | 日期：YYYY-MM-DD
> 来源：brief.md v1

## 故事大纲（投稿用，500–1500 字）

### 核心设定
<世界观、时代、核心矛盾，2–3 段>

### 主要人物
<每人 2–3 句：身份、欲望、与主角关系>

### 故事梗概
<按起承转合写完整主线，含结局方向>

### 差异化与卖点
<编辑最关心的 2–3 个看点>

## 分集大纲

> 默认 30 集；用户指定集数时按指定执行。
> 每集：开场钩子 + 核心事件 + 结尾卡点，各 1–2 句，可拍摄。

| 集数 | 开场钩子 | 核心事件 | 结尾卡点 |
|------|----------|----------|----------|
| 第1集 | | | |
| 第2集 | | | |
| ... | | | |

## 节奏自检

| 阶段 | 集数 | 本剧安排 | 是否达标 |
|------|------|----------|----------|
| 抓人 | 1–10 | | |
| 升级 | 11–30 | | |
| 高潮 | 31+ | | 如不足 30 集标 N/A |

## 逻辑链验证
1. 触发 → ... → 结局（用「因此/但是」串联，5–8 步）

## 风险提示
<沿用 brief 并补充大纲阶段新发现的风险>

## 下一步
- [ ] 用户确认大纲
- [ ] 进入 `drama-episode` 写分集剧本
```

## 生成规则

### 1. 忠于 brief

- 人物、反差、核心事件链与 brief 一致
- 不得擅自改结局方向或主角人设

### 2. 分集大纲

- **第 1 集**：前 30 秒级冲突必须具体（对谁、在哪、发生什么）
- **每集结尾**：必须有可拍摄的卡点（悬念 / 反转 / 情绪峰值）
- **信息密度**：每集只推进 1–2 个关键情节点，不堆砌
- **前 3 集**：与 brief 中的钩子表对齐，可细化不可推翻

### 3. 节奏

- 1–10 集：立人设、建冲突、小胜或小挫
- 11–30 集：矛盾升级、副线收束、为高潮铺垫
- 30 集体量：中点转折约在 12–15 集，至暗时刻约在 22–25 集

### 4. 平台对齐

对照 `hongguo-platform`：主线清晰、开场不慢热、格式适合后续扩写为场次剧本。

### 5. 防呆规则（生成时遵守，减少 qc 必改项）

| 规则 | 要求 |
|------|------|
| 赛场/时间线 | 多场比赛须写「赛场时间线」表，标注每场性质（资格赛/分站/总决赛）与结果 |
| 梗概与分集一致 | 故事梗概的赛果须与分集大纲末集卡点一致（如「险胜」≠「亚军」） |
| 泄密/反转不重复 | 同类事件分阶段升级（草图→核心数据），不写两次相同泄密 |
| 资源/道具铺垫 | 众筹零件、关键道具须在前文 3–5 集有可拍摄铺垫 |
| 反派作恶合规 | 砸车间/陷害后须在同集或下集接「报警/行业封杀」，不写私刑 |
| 伏笔回收 | 干扰器、道具等伏笔须在 3 集内回收或标注回收集 |

## 工作流

```
1. 读取 brief.md
2. 提取：梗概、人物、事件链、前 3 集钩子、情绪节奏
3. 写故事大纲（编辑可读，说明白卖点）
4. 展开分集大纲（默认 30 集），遵守防呆规则
5. 自检：每集有卡点、前 3 集够强、逻辑链通顺、赛场时间线表
6. 写入 outline.md
7. 按 drama-qc 自动修复规则做一轮自检修订（无需用户确认）
8. 呈现给用户，不自动进入分集剧本
```

## 示例要点

**brief**：王牌保镖伪装保姆潜入豪门

**第 1 集卡点**：霸总捡起她「失手」掉出的弹壳，眼神变了

**分集大纲第 1 集**应细化为可拍摄事件，而非「两人产生误会」这类空话。
